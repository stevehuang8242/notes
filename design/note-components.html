00 Page Scaffold

頁面骨架 + meta

每個 note 檔都從這個 <head> 開始：FOUC 防呆 script、Saira 字體、整段 token :root（複製本檔 §1 TOKENS）。note:* meta 依統一慣例填寫——src/date/title/dek 必填，date 用 YYYY.MM.DD，people 是署名（原創類放單一角度標籤或留空）。寫完用 node validate-notes.js <file> 自查。

01 Hero

靠左 ticker 開場

全站單一 hero：靠左，代號標籤（.ticker 內 .tk badge + .mt meta）+ 細粗混排多行標題 + 副標 +（選填）CTA。標題強調三選一：.hl 藍色（--accent，一般重點預設用這個）、<u> 綠色（--pos，限真有正向語意如「現金引擎」）、.xline 中性底線；再加 .thin 做細粗反差。副標可內嵌 <b>／.c／.b。分析型常接 Metric Strip，敘事型常接 CTA。CTA 僅作頁內跳轉（#anchor）；podcast／影片的 YT 來源連結不放 hero，一律收在 footer .meta（見 19 · Footer）。

13 SOTP

分類加總

每列：名稱（可染 .c/.b）+ 描述 + 區間估值。用在 sum-of-the-parts、預算拆分等「逐塊列出 + 各自一句」的場合。

14 Bull / Bear

多空兩面

兩欄對照（.bbc.bull / .bbc.bear）。標題一律中性灰，多空只靠 ▲／▼ 區分——語意色僅留在內文 <strong>。任何「正反兩面」都適用，不限財務（如「樂觀情境 vs 風險情境」）。

▲ Bull Case

正面論點放這裡，關鍵假設用 strong 提亮。標題維持中性灰。

▼ Bear Case

反面論點放這裡，對稱呈現。靠 ▼ 與內文區分。

15 Callout

引述框

左色條強調塊：中性（白）/ .c（正）/ .b（負）。用來框住「核對結論」「一句話總結」這種需要被看見的句子。

中性 callout：一句話總結或關鍵核對放這裡。

正向 callout：左條轉綠，用於好消息／cash engine。

負向 callout：左條轉紅，用於風險／燒錢核對。

16 Glossary

詞彙浮層

在內文把專有名詞包成 <span class="gl" data-d="解釋…">詞</span>：虛線底線，hover（或手機點擊）浮出解釋。讓行文保持精煉、又不犧牲可讀性。

例如解釋 EBITDA 或 sum-of-the-parts，讀者把游標移上去就看得到，不打斷句子。

19 Footer

頁尾

大字結語（.big，可帶 small 副句）+ meta caps（來源／日期／與談人／連結）+ 免責 .disc。每篇都該有免責那行，尤其涉及投資觀點。podcast／影片來源的 YT 連結就放在 .meta（緊接來源·日期·與談人）——這是來源連結的唯一正規位置，hero 與內文不重複放。

「最後一句有重量的結語放這裡。」可以補一句副標收尾。
